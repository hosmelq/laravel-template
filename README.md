# Laravel Template

Laravel Template is a starting point for Laravel applications with PostgreSQL, Sentry, strict development tooling, and optional features installed through an interactive command.

## Introduction

The base application includes the dependencies and conventions used by every project. Application-specific features are kept in patches and can be installed together in any combination.

## Requirements

- Composer
- Git
- Nub
- PHP 8.5 or higher
- PostgreSQL

## Installation

First, create a new repository from this template and clone it locally.

Next, prepare the environment files and configure the PostgreSQL connections:

```bash
cp .env.example .env
cp .env.testing.example .env.testing
```

Then, install the base application:

```bash
composer setup
```

Finally, run the interactive installer and select the features required by the application:

```bash
php artisan template:install
php artisan migrate
```

## Optional Features

The installer currently provides:

- **Authentication** — Laravel Fortify authentication and two-factor authentication.
- **Frontend** — Inertia, React, HeroUI, Tailwind CSS, Vite, and frontend tooling.
- **Health** — Application health checks, the health endpoint, and scheduled checks.
- **Media** — Media Library, image processing, and S3 filesystem support.
- **Organizations** — Organizations, memberships, ownership, policies, and current-organization support.

Selected patches are checked and applied as a single group. The installer then updates Composer dependencies, installs frontend dependencies when needed, and runs `composer agent:setup`.

Generated files such as `composer.lock`, `pnpm-lock.yaml`, and agent configuration are not stored in the patches; they are regenerated by their respective installation commands.

Features that are already installed are skipped.

After a successful installation, the installer command, patches, related tests, and feature-combination script are removed. Select every feature the application needs during the initial run.

## Development

When the frontend feature is installed, start the Vite development server with:

```bash
nub run dev
```

Create a production build with:

```bash
nub run build
```

## Testing

Run the test suite with:

```bash
composer test
```

Test every optional feature combination in an isolated temporary checkout with:

```bash
scripts/test-template-feature-combinations
```
